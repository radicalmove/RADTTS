    Set up helper app

    Run this once on each helper computer. After setup, audio jobs can process on that device automatically.

    Preparing setup options...

    Windows installer Mac installer

    Step-by-step setup

    1. Open RADTTS in your browser and select your project.
    2. Click Set up helper app.
    3. Choose Windows installer or Mac installer and run it.
    4. If your browser blocks downloads, use Copy Mac setup command or Copy Linux setup command and run it in Terminal.
    5. Approve any security prompts once, then allow the helper app to run in the background.
    6. Return to RADTTS and click Refresh helper status to confirm it is online.

    Troubleshooting

    • If installer download does not start, copy and run the setup command manually.
    • If macOS blocks the app, go to System Settings > Privacy & Security and allow it, then run again.
    • If status stays offline, close and reopen the browser tab, then refresh helper status.
    • If command install fails, check Python is installed (`python3 --version`) and try again.
    • If helper count still does not increase, ask an admin to regenerate setup links and retry.

    Help & Guide

    RADTTS turns script text into project-based voice outputs using either a reference sample or a built-in voice.

    Read this first

    1. Create or open the project you want to work in.
    2. Choose a voice source, then enter or load the script text.
    3. Set the voice and output options you need for this run.
    4. Generate audio and wait for the finished version to appear in Completed versions.
    5. Review the versions, keep the best result, and save the output you want to use elsewhere.

    Tip

    Read the overview first so you can choose the right voice source before you start editing the script or trimming reference audio.

    Note

    Each run is saved in the current project, so you can compare versions without losing earlier outputs or redoing earlier settings.

    Use this flow when you want to turn a script into a new spoken version.

    Fast path

    1. Create or open a project before you start so the script, samples, and outputs stay together.
    2. Paste text into the script box or load it from a supported file.
    3. Choose the voice source, quality, transcript, and output format for this run.
    4. Click Generate audio and wait for the progress panel to complete.
    5. Play the finished versions in the output list and compare them before deciding which one to keep.

    Note

    If you edit the script or change the voice source, generate a new version rather than expecting an old output to update.

    Tip

    Keep one browser tab on the project page while generation runs so you can catch the finished version as soon as it lands.

    Choose between the built-in Qwen voice and a cloned reference sample for this workflow.

    Choosing the source

    1. Open the project where you want to generate audio.
    2. Choose Built-in Qwen voice if you do not want to clone from a sample.
    3. Choose Clone from sample when you want the output to follow the voice in a reference recording.
    4. Run a short version first, then generate the full script once the selected voice sounds right.

    Note

    If the voice sounds wrong, switch the voice source or replace the reference sample rather than looking for a separate custom voice setting.

    Tip

    Use a clean reference sample or the built-in voice that best matches the result you want before you commit to a longer run.

    A good reference sample gives RADTTS a clearer voice to copy and usually leads to more natural results.

    Sample quality checklist

    1. Choose a clean recording with one speaker, steady volume, and as little background noise as possible.
    2. Trim or select a useful sample that sounds representative of the voice you want to clone.
    3. Preview the sample and make sure it does not include long silences, music, or other speakers.
    4. Use that sample for generation and replace it if the voice output sounds unstable or off-target.

    Troubleshooting

    Very short, noisy, or mixed-speaker samples usually produce weaker voice matches and often need to be replaced before the next run.

    Note

    If the sample is usable but not ideal, trim it to the strongest section first instead of making the whole clip longer.

    Completed versions let you compare runs, check metadata, and keep the output that best matches the script.

    Review workflow

    1. Open Completed versions after each generation run.
    2. Use playback controls and the version metadata to compare timing, settings, and output details.
    3. Download or save the version you want to keep once you are happy with it.
    4. Use the project output folder path when you need to find the generated files outside the app.

    Note

    Older versions stay available in the project until you remove them, so label and review carefully if you generate many runs.

    Tip

    Keep the best version name visible in the list so you can spot the right file even after several rounds of iteration.

    Helper processing sends generation work to a live RADTTS helper when one is available.

    When helpers matter

    1. Check the helper status row before you start a longer generation run.
    2. Use Set up helper app if you need to connect another machine for processing.
    3. Start the job and watch the progress panel for the machine handling the work.
    4. If no helper is live, the job may wait for helper assignment before falling back to the main RADTTS machine.

    Note

    If the helper is offline, your job may wait for assignment before it falls back to the main RADTTS machine.

    Tip

    A helper must stay online for jobs to keep moving, so refresh status if the device was recently restarted or disconnected.

    Use these checks when generation is slow, the helper is unavailable, or the output does not sound right.

    Common recovery steps

    1. If a job is taking a long time, leave the page open and watch the stage and ETA before retrying.
    2. If helper status is offline or unknown, refresh it and expect the job to run without helper support.
    3. If generation fails, recheck the active project, the script text, and the selected voice source.
    4. If the voice sounds wrong, swap in a cleaner reference sample or switch to the built-in voice.
    5. If the reference sample is not loading, choose it again or use a saved sample from the current project.

    Troubleshooting

    Most generation problems come from missing text, an invalid voice source, a weak reference sample, or a helper that dropped offline mid-run.

    Note

    If the app keeps reporting a stalled helper, switch to the built-in voice for the next run so you can keep moving.
